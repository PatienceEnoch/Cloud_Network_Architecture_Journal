# Cloud Fundamentals

This document covers core cloud computing concepts relevant to CompTIA Network+ and real-world networking environments.

---

## What is Cloud Computing?

Cloud computing is the delivery of computing resources over the internet, including servers, storage, networking, databases, and applications.

Instead of purchasing and maintaining physical hardware, organizations rent resources from a cloud provider.

---

## Cloud Service Models

### Infrastructure as a Service (IaaS)
Provides virtualized computing resources over the internet.

- You manage: operating system, applications, and data
- Provider manages: physical hardware and infrastructure

Examples:
- AWS EC2
- Azure Virtual Machines

---

### Platform as a Service (PaaS)
Provides a platform for developing, running, and managing applications.

- You manage: applications and data
- Provider manages: operating system, runtime, and infrastructure

Examples:
- Azure App Services
- Google App Engine

---

### Software as a Service (SaaS)
Provides fully managed software applications over the internet.

- You manage: usage and access
- Provider manages: everything else

Examples:
- Microsoft 365
- Gmail
- Dropbox

---

## Cloud Deployment Models

### Public Cloud
Infrastructure is owned and operated by a third-party provider and delivered over the internet.

Examples: AWS, Azure

---

### Private Cloud
Infrastructure is dedicated to a single organization, either on-premises or hosted.

Provides greater control and security.

---

### Hybrid Cloud
Combines public and private cloud environments, allowing data and applications to move between them.

---

### Community Cloud
Shared infrastructure used by organizations with similar requirements or regulatory needs.

---

## Key Cloud Characteristics

- On-demand self-service: Resources can be provisioned without human interaction
- Broad network access: Services are available over the network via standard mechanisms
- Resource pooling: Provider resources are shared across multiple customers
- Rapid elasticity: Resources can scale up or down quickly
- Measured service: Usage is monitored and billed accordingly

---

## Benefits of Cloud Computing

- Scalability and flexibility
- Reduced capital expenditure
- High availability and redundancy
- Built-in disaster recovery options
- Global accessibility

---

## Risks and Considerations

- Data security and privacy concerns
- Vendor lock-in
- Dependence on internet connectivity
- Misunderstanding of responsibility boundaries

---

## Shared Responsibility Model

### Cloud Provider Responsibilities
- Physical data centers
- Hardware and infrastructure
- Core networking

### Customer Responsibilities
- Data protection
- Application configuration
- Identity and access management
- Security settings

---

## Real-World Example

A company hosts a web application using virtual servers in the cloud (IaaS), stores files in cloud storage, and uses a content delivery network (CDN) to improve performance for users in different geographic regions.

---

## Summary

Cloud computing enables organizations to use scalable, on-demand computing resources without owning physical infrastructure, improving efficiency and reducing costs.
